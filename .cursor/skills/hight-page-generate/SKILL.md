# 高级页面生成（High-End Page Generator）

## 工作流总览

1. **先定设计系统** — 输出色板、字号阶梯、间距刻度、圆角规范
2. **翻译感觉为技术词** — 把"高级"转译为 CSS/动效/3D/组件维度的具体关键词
3. **四层质量检查** — CSS 材质感 → 动效呼吸 → 3D 空间 → 组件系统，逐层落地
4. **参考 + 约束** — 给方向与边界，稳定输出
5. **小步迭代** — 先改一维，确认后扩展到全局

---

## 第一步：先定设计系统

**不要上来就写页面结构，先让用户确立设计系统。**

请用户确认或共同拟定以下四大类：

| 模块 | 内容 |
|------|------|
| **色彩系统** | 主色、辅色、中性色（背景/边框/文字）、强调色；是否使用渐变 |
| **字体系统** | 字体族、层级（标题/正文/辅助）、字号阶梯、字重、行高 |
| **布局系统** | 间距刻度（spacing scale）、圆角规范、内容区 max-width、栅格列数 |
| **组件系统** | 按钮（主/次/轮廓）、卡片、表单输入、导航、弹窗等基础样式 |

**写法示例：**
> Inter 字体；标题 32px Bold / 正文 16px Regular / 辅助 13px；行高 1.5；间距刻度 4-8-12-16-24-32-48-64；圆角 8px（卡片）/ 6px（按钮）/ 4px（输入框）；深色背景主色 #10B981，中性灰阶 #0A0A0A → #FAFAFA。

**关键原则：** 后续修改不符合预期的界面时，优先指出"违反了哪条设计系统规则"，而不是只说"不好看"。

---

## 第二步：把抽象感觉翻译为技术词

用户说的"更高级"、"更有设计感"、"更有科技感" — **必须翻译为具体可实现的技术描述**，否则 AI 会输出保守平均值。

**负向示例 → 正向修正：**

| 用户说的 | 应翻译为 |
|---------|---------|
| "更高级一点" | 增加 backdrop-blur 玻璃质感、subtle border 细边框、soft shadow 轻阴影 |
| "更有设计感" | bento grid 分块布局、stagger animation 错落出现、信息层级分明 |
| "更好看" | 明确具体维度：配色、字体、间距、组件状态、动效 |
| "更有科技感" | 深色底 + 渐变网格背景 + 微光描边 + 粒子场/3D 点缀 |

**核心公式：** 抽象形容词 → 具体技术词（关键词速查见 `references/keywords.md`）

---

## 第三步：四层质量模型

按从底到顶的次序，逐层检视页面质量。建议每层做完再做下一层。

### 第一层：CSS 材质感（占高级感 ~60%）

**公式：** 材质感 + 空间感 + 层次感 = 看起来「贵」

核心 CSS 关键词（选 3-5 个写入 Prompt）：
```
backdrop-blur  soft-shadow  subtle-border  gradient-mesh
noise-texture  bento-grid   fluid-layout   sticky-section
scroll-snap
```

### 第二层：动效与呼吸感

**原则：** 动效不是炫技，而是自然、克制、有目的。

核心动效关键词：
```
micro-interactions  hover-feedback  scroll-reveal
stagger-animation   smooth-transition  easing-curve
parallax  spring-animation
```

**工具选型默认组合：** Motion（Framer Motion）— 轻量现代，适合微交互与页面转场。仅在需要复杂时间线控制时切换到 GSAP。

### 第三层：3D 与空间感

**原则：** 3D 烘托氛围，不抢主内容。

核心 3D 关键词：
```
particle-field  3D-orb  shader-gradient
floating-geometry  webgl-background  camera-parallax
interactive-canvas
```

**约束（写进 Prompt）：** 作为背景，不遮挡正文与主 CTA；低面数、稳定帧率。

**技术栈默认：** Three.js / React Three Fiber（R3F）。

**注意：** 默认不放重 3D。仅在页面类型适合（品牌站、产品展示页等）时启用。

### 第四层：组件系统与产品感

**原则：** 页面不是零件拼贴，而是统一系统。

系统向关键词：
```
design-system  component-variants  shadcn/ui
tailwind-css   dashboard-layout    modal  toast  empty-state
```

**对 AI 的四条明确要求：**
1. 组件视觉与交互语言统一
2. 状态完整（默认/悬停/点击/禁用/加载/空状态）
3. 标题/正文/辅助信息层级分明
4. 桌面/平板/手机适配到位

---

## 第四步：参考 + 约束

### 参考图（Reference）

给 AI 方向：
- 可提供截图、链接、竞品页面
- 让 AI 对齐整体气质、布局逻辑、信息层级
- 用参考图做拆解 → 要求 AI 输出色板推测、字体层级表、布局结构

### 约束（Constraints）

"不要做什么"往往比"要做什么"更有效。

示例约束：
- 保持参考的版式节奏与色彩气质，但文案与组件内容为我的业务
- 固定为 Dark mode
- 禁止大面积渐变背景、Emoji、彩虹色按钮、大圆角堆砌
- 禁止每块都用重阴影

---

## 第五步：小步迭代

1. **先改一个维度** — 只改主色、只改字体、只改 spacing
2. **确认方向** — 满意后让 AI 把同一规则扩展到全页/全组件
3. **逐步加细** — 动效、状态（hover/disabled/loading）、空状态、错误提示
4. **统一检查** — 对照设计系统做一致性扫尾（圆角、阴影、间距刻度是否混用）

## 常见错误模式

- **只给抽象形容词** — "更高级" → 需要翻译为具体技术词
- **上来就写页面** — 应先定设计系统再写结构
- **一次性大改** — 应先改一维、再扩展
- **忘了加约束** — 不加约束 AI 易出"平均分配"式模板
- **忽略了组件系统** — 只有前三层没第四层，页面像 Demo 不像产品
